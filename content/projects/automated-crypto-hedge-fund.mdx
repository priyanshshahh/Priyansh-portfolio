## Executive Summary

Project CHF is a reproducible, leakage-safe quantitative cryptocurrency research pipeline (frozen `v1.0-research-release`). It tests whether market and on-chain features can construct crypto portfolios outperforming BTC, ETH, BTC/ETH 50-50, and equal-weight universes after costs and realistic validation. The headline result is a deliberate negative: `alpha_verified = false`. Source: [github.com/priyanshshahh/chf-v2](https://github.com/priyanshshahh/chf-v2).

## Problem & Constraints

A system designed to find alpha is easy to fool. CHF is built to reject unsupported alpha claims - surfacing candidates, forcing deterministic allocation, applying transaction costs, and comparing against four benchmarks before any alpha declaration.

<Callout title="Architecture" variant="architecture">
UniverseAgent → MarketDataAgent → OnChainAgent → FeatureAgent → LabelAgent → ModelAgent → AlphaResearchAgent → PortfolioAgent → BacktestAgent. Stages communicate only through file artifacts under `data/` (Parquet/JSON).
</Callout>

## Methodology

- Purged + embargoed walk-forward signal screening with Optuna hyperparameter tuning
- Leakage-safe feature generation and exact forward calendar log-return labels (7/14/30d)
- Deterministic prediction-only portfolio allocation (HRP, Top-K, vol-scaled)
- Transaction-cost-aware backtesting with benchmark sanity checks
- 27 test files, MLflow logging, content-hash reproducibility (`CHF_SEED=42`)
