## Executive Summary

ClaimGuard-AI is an agentic pre-submission financial risk engine for healthcare revenue cycle management, submitted for the AIxBio Hackathon. It replaces static rule-based scrubbers with a three-layer intelligence stack: agentic extraction, quantitative scoring, and financial optimization.

## Problem & Constraints

Healthcare claims face denial risks from documentation gaps, procedure mismatches, and payer policy violations. Manual scrubbing is slow and misses expected financial loss prioritization.

<Callout title="Architecture" variant="architecture">
Physician Note + CPT/ICD/Payer → Nebius Gemma JSON extraction → XGBoost denial probability → Expected Loss EL = V × P → DuckDB knapsack sort → Next.js auditor worklist.
</Callout>

## Methodology

- Nebius Token Factory (Gemma 3 27B) extracts strict JSON clinical flags (Pydantic strict=True)
- XGBoost classifier predicts denial probability with scale_pos_weight for imbalance
- DuckDB analytical sort with bounded knapsack (capacity K/day) prioritizes by expected financial loss
- Next.js frontend: Dashboard, Claims Queue, Agent Studio, Reports with PDF/CSV export
