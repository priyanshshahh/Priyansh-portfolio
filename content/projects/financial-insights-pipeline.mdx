## Executive Summary

The Financial Insights Pipeline (FIP) creates a reproducible, open pipeline to automatically gather institutional equity holdings from ARK Invest daily ETF CSVs and SEC EDGAR Form 13F filings, normalize them into a single schema, and calculate position-change events (NEW, INCREASE, DECREASE, EXIT).

## Problem & Constraints

Holdings data are inconsistent (tag variants, malformed tables), fragmented (XML/HTML/TXT/CSV), and lagged (13F quarterly). Manual workflows are error-prone and non-replicable. FIP eliminates manual scraping for ML feature engineering and academic research.

<Callout title="Architecture" variant="architecture">
Scheduler → Source Discovery → Downloader → Parsers → Validation → Transformer → Diff Engine → PostgreSQL + Parquet → REST API → Dashboard.
</Callout>

## Methodology

- Ingested SEC EDGAR 13F-HR/A via company-submissions JSON and ARK ETF daily CSVs (ARKK, ARKG, ARKQ, ARKW, ARKX)
- Normalized to unified schema with CUSIP/ticker mapping and amendment supersession handling
- Implemented deterministic change-detection: NEW, EXIT, INCREASE, DECREASE with configurable thresholds
- Built compliance-aware ingestion with rate limiting, exponential backoff, and auditable lineage
