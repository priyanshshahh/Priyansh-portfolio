## Executive Summary

This research project engineers an end-to-end pipeline that extracts institutional-grade equity data from WRDS, integrates Perplexity and Gemini APIs for portfolio construction, and evaluates prompt-engineered investment strategies through rigorous backtesting and index compliance validation.

## Problem & Constraints

Traditional portfolio construction relies on static factor models. This project explores whether large language models can synthesize heterogeneous financial signals into actionable allocation decisions while maintaining index compliance, risk constraints, and reproducible evaluation methodology.

<Callout title="Architecture" variant="architecture">
The system follows a three-stage pipeline: data extraction (SQL/WRDS), LLM signal generation (Perplexity + Gemini), and portfolio validation with rebalancing simulation.
</Callout>

## Methodology

- Built SQL pipelines to extract and clean WRDS equity fundamentals and returns data
- Designed prompt templates for sector allocation, risk budgeting, and rebalancing decisions
- Implemented Portfolio Validation and Cleaning framework ensuring complete index compliance
- Executed Return Computation and Portfolio Rebalancing with drawdown-aware risk rules
- Authored URECA research paper evaluating systematic Tech sector concentration patterns
